1. La idea central del proyecto

Este proyecto es un sistema de análisis que reinterpreta el gasto público nominal como impacto económico real, cruzando tres dimensiones que hoy se observan de forma separada:

finanzas públicas (ejecución presupuestaria),

inflación,

estacionalidad.

Utiliza exclusivamente datos públicos, no personales, publicados por organismos oficiales, y no pretende operar en tiempo real.

El problema económico que aborda es simple pero profundo:
en contextos inflacionarios, el gasto público se comunica y se evalúa en números que no representan su efecto real sobre la economía.

El proyecto existe porque hoy:

el gasto se mide en pesos corrientes,

la inflación erosiona silenciosamente su poder de compra,

la estacionalidad distorsiona comparaciones,

y el impacto económico real queda oculto.

El valor del sistema no está en descubrir datos nuevos, sino en darle significado económico real a datos que ya existen.

2. El problema explicado sin técnica

Mirar gasto público en términos nominales es engañoso porque los números crecen aunque la capacidad de compra caiga.

Un presupuesto puede:

duplicarse en pesos,

y aun así comprar menos bienes, menos servicios o menos horas de trabajo.

La inflación rompe la comparación en el tiempo.
La estacionalidad rompe la comparación entre meses.

Cuando estos factores no se cruzan:

un aumento puede ser una pérdida,

un pico puede ser normal,

una “eficiencia” puede ser solo un efecto calendario.

Hoy se concluye mal porque:

se comparan montos sin contexto,

se interpretan variaciones nominales como mejoras reales,

se discuten números que no representan impacto económico.

El problema no es la falta de datos.
El problema es la falta de una lectura económica integrada y consistente.

3. Qué hace y qué NO hace el proyecto
Qué hace el proyecto

El proyecto responde preguntas de este tipo:

¿Este gasto hoy compra más, menos o lo mismo que antes?

¿El aumento presupuestario compensó la inflación o no?

¿Esta variación es real o es estacional?

¿Qué parte del gasto perdió poder adquisitivo de forma sostenida?

Las respuestas que da son:

comparativas,

cuantificables,

reproducibles,

explicables.

Qué NO hace el proyecto

El proyecto no emite juicios de valor.

No responde:

si el gasto está “bien” o “mal”,

si es eficiente o ineficiente,

si hay corrupción,

si el impacto social es positivo o negativo.

No recomienda políticas públicas.
No predice el futuro.
No acusa ni justifica decisiones.

El límite conceptual es explícito:

Este proyecto no evalúa decisiones políticas.
Evalúa cómo cambia el poder de compra del gasto en el tiempo.

4. El concepto de “evento” del sistema

En este proyecto, un evento no es una transacción.

No hay compras en tiempo real.
No hay pagos en streaming.
No hay movimiento instantáneo de dinero.

Un evento es un cambio publicado oficialmente que altera el significado económico del gasto ya ejecutado o por ejecutar.

El sistema reacciona a publicaciones oficiales porque:

son el único reflejo legítimo del gasto público,

son auditables,

son reproducibles,

y fijan el marco económico real.

Los 5 eventos que el sistema escucha

Publicación de nuevos datos de ejecución presupuestaria
Cambia cuánto gasto efectivamente ocurrió y cuándo.

Revisión o corrección de datos presupuestarios históricos
Cambia la base sobre la que se comparan períodos anteriores.

Publicación de un nuevo índice de inflación
Cambia el poder de compra real de todo el gasto pasado.

Actualización del calendario fiscal o cierre de períodos
Cambia la interpretación estacional de ciertos picos o caídas.

Cambio en la estructura presupuestaria publicada oficialmente
Cambia cómo deben agruparse y compararse partidas en el tiempo.

Todos estos eventos:

son detectables automáticamente,

no requieren intervención humana,

alteran el significado económico del gasto.

5. Cómo el sistema “escucha” sin tiempo real

El principio operativo del proyecto es:

observar → versionar → comparar → recalcular

El sistema:

observa publicaciones oficiales,

versiona cada estado del dato,

compara contra estados anteriores,

recalcula el impacto económico real.

Este enfoque es deliberado.

No prometer tiempo real es:

más honesto (el gasto público no ocurre en tiempo real),

más defendible (se basa en datos oficiales cerrados),

más valioso (permite análisis correctos, no instantáneos).

Prometer streaming sería:

conceptualmente falso,

económicamente irrelevante,

y analíticamente engañoso.

6. Qué tipo de producto es (y cuál no)

Este proyecto no es solo un dashboard, porque no se limita a mostrar datos crudos.

Es un sistema de reinterpretación económica:

cruza variables que hoy se leen separadas,

produce comparaciones que no existen de forma estándar,

revela cambios reales donde antes había ilusión nominal.

No es un informe estático, porque:

los datos cambian,

la inflación reescribe el pasado,

el significado económico se actualiza con cada publicación.

Puede exponerse como API porque:

las respuestas no dependen del “ahora”,

sino del estado económico actualizado del dato,

y son válidas en cualquier momento.

7. Límites, riesgos y supuestos
Límites del proyecto

El proyecto pierde valor cuando:

la inflación es baja y estable por largos períodos,

los datos públicos son discontinuos o poco confiables,

se pretende usarlo para juzgar impacto social o político.

Supuestos económicos

El índice de inflación elegido es una aproximación válida.

El gasto puede compararse en términos reales aunque su composición cambie.

La estacionalidad histórica es relevante para interpretar el presente.

Interpretaciones incorrectas o peligrosas

Sería incorrecto:

usar el sistema para acusar corrupción,

afirmar eficiencia o ineficiencia final,

comparar impacto social entre programas distintos.

El proyecto no reemplaza análisis cualitativos.
Los precede y los ordena.

8. Definición final del proyecto

Definición corta

Un sistema que traduce el gasto público nominal en impacto económico real, ajustado por inflación y estacionalidad, usando solo datos públicos.

Frase ancla

El presupuesto puede crecer en pesos y achicarse en realidad.
Este proyecto existe para mostrar esa diferencia.
